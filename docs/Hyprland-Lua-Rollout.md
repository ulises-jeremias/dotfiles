# Hyprland Lua Migration — Rollout Guide

> Part of `ulises-jeremias/dotfiles#179` — closes `ulises-jeremias/dotfiles#188`

## Prerequisites

- Hyprland >= 0.55 installed
- chezmoi configured with this dotfiles repo
- Migration branch reviewed and validated against the current `main` baseline

## PR merge order

The sub-issues were designed to be merged in order, but are intentionally
independent. They can be reviewed in any order. For best results, merge:

1. `#181` — Entrypoint + static settings (creates `hyprland.lua`, modules, stubs)
2. `#182` — Keybindings + keyboard help
3. `#183` — Window rules
4. `#184` — Animation profiles + rice switching
5. `#185` — Smart Colors Lua output
6. `#186` — Runtime integrations audit (doc-only)
7. `#187` — Documentation update
8. `#188` — This rollout guide

## Apply to host

After the migration branch is validated:

```sh
# 1. Get latest config
chezmoi update

# 2. Preview changes
chezmoi diff

# 3. Apply
chezmoi apply

# 4. Reload Hyprland
hyprctl reload
```

## Verify

After reload, confirm:

- [ ] Quickshell bar appears
- [ ] Terminal opens (`SUPER + Return`)
- [ ] Workspace switching works (`SUPER + 1,2,3`)
- [ ] Application launcher opens (`CTRL + Space`)
- [ ] Window rules apply (e.g., `pavucontrol` floats)
- [ ] Smart Colors border colors match current theme
- [ ] Animation profile switching works via rice change
- [ ] Game Mode toggle still works
- [ ] Keyboard help (`SUPER + /`) shows keybindings

## Rollback

If `hyprctl reload` fails or the desktop is unusable:

1. Use known-good HorneroConfig binds (`SUPER + SHIFT + Q` = close window,
   `CTRL + Space` = launcher, `SUPER + SHIFT + E` = exit) to open a terminal.
2. Roll back to the previous known-good source state and re-apply:

```sh
cd ~/.dotfiles
git switch main
git pull --ff-only
chezmoi apply --source=. -c ~/.config/chezmoi/dotfiles.toml
hyprctl reload
```

Note: if Hyprland was originally started from `hyprland.conf`, the running
session may regenerate a tiny autogenerated `hyprland.conf` stub until the next
login. That file is not HorneroConfig's legacy configuration; `hyprland.conf.d/`
is removed by this migration, and a fresh Hyprland start with `hyprland.lua`
present will use the Lua entrypoint.

Or, for an immediate live-session fallback without changing git state:

```sh
mv ~/.config/hypr/hyprland.lua ~/.config/hypr/hyprland.lua.bak
hyprctl reload
```

3. If all else fails and you need an immediate live-session fallback, disable
   the Lua entrypoint temporarily. This only helps if you have a separate
   backup of the legacy hyprlang files, because HorneroConfig removes them
   during this migration.

```sh
# Temporarily rename the Lua entrypoint
mv ~/.config/hypr/hyprland.lua ~/.config/hypr/hyprland.lua.bak
# hyprland 0.55+ will fall back to hyprlang if no .lua is found
hyprctl reload
```

## Validation checks

| Check | Command | Expected |
|-------|---------|----------|
| Lua entrypoint exists | `test -f ~/.config/hypr/hyprland.lua` | `true` |
| Modules directory exists | `test -d ~/.config/hypr/hyprland.lua.d` | `true` |
| Keybindings load | `hyprctl binds` | Lists all binds |
| Bind count baseline | `hyprctl binds -j \| jq 'length'` | Not emergency-only (`> 100`) |
| Monitor config | `hyprctl monitors` | Monitors configured |
| Animations active | `hyprctl getoption animations:enabled -j` | enabled |
| Legacy conf removed | `test ! -f ~/.config/hypr/hyprland.conf` | `true` |
| Legacy conf.d removed | `test ! -d ~/.config/hypr/hyprland.conf.d` | `true` |
